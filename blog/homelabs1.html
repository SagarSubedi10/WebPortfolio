Home Labs - Web Server Administration



Here is a list of novice level tasks that I completed when I was first beggining to learn Web Server Administration. A good starting point for someone who is looking to begin in this field as well.

  • Install Windows Server (08, 12,16, 19) using VirtualBox
    • Install Abyss Web Server and IIS here. Make sure localhost displays your webpage
  • Apache Web Server
    • Host website using Apache Web Server on Linux Server by Installing Ubuntu or OpenSuse in VirtualBox
      • You will have to install and configure Apache
      • SU (switch users)
      • Rcapache2 start (to start the apache service)
      • Insserv apache2 (to start apache automatically when you restart the server)
      • Browse to the /srv/www/htdocs/index.htm directory to modify the page
    • Congratulations you have an Apache Web Server running inside your Linux Server
  • Host Website by a URL (DNS)
    • Install DNS if it is not already installed and then configure the DNS with the records
      • Yourdomain.com ----- host(A) ----- Your IP
      • ns. ----------------------- host(A) ---- Your IP
    • Modify bindings in the webserver to have IP the same as configured in the IIS and domain same as what was created in the DNS as a record (Yourdomain.com)
    • Go to the created domain instead of using localhost!
  • Active Directory and Intranets
    • Install active directory within your windows servers and configure your domain controller
    • Create a new site in the IIS and create the HTML file
  • Apache on Windows
    • Note your IP address and install apache for windows from the internet. Configure apache to run on port 8080 or 8000, whichever is not in use. Find the directory for Apache in Windows. It should be running on localhost:8000 or localhost:8080
  • Secure the Web Environment
    • Manage firewall by doing things like creating new outbound or inbound rules to block or allow ports. Example, you can create a new outbound rule to block port “xxx” to block Skype connections.
    • You can also secure the web environment by going into the control panel and checking for windows updates.
    • You can utilize the Best Practices Analyzer in your server manager console by clicking on Run a BPA scan.
  • Monitor Web Server
    • Within event viewer you can create a custom view to show all errors that have occurred.
    • The resource monitor will show you the health of your CPU, Memory, ETC
    • You can add IIS counters to the performance monitors
